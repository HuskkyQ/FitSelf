# FitSelf — iOS健身数据追踪与饮食管理App 功能设计文档

## 1. 产品概览

### 1.1 产品定位

**FitSelf** 是一款iOS原生健身数据追踪与饮食管理应用，面向健身新手和有经验的健身者。以数据驱动为核心设计理念，让用户「看见自己的变化」。

### 1.2 设计原则

| 原则 | 说明 |
|------|------|
| 本地优先 | 所有数据本地存储，iCloud仅做同步，无需服务器 |
| 离线可用 | 无网络环境可完整使用所有功能 |
| 隐私优先 | 健康数据不上传任何第三方服务器 |
| 新手友好 | 渐进式功能展示，新手不被复杂数据淹没 |

### 1.3 商业模式

完全免费，无内购、无广告、无订阅。

### 1.4 目标用户

- **健身新手**：需要引导和基础数据追踪，了解自己的身体状况
- **有经验健身者**：需要精细化数据分析、营养配比优化

## 2. 技术架构

### 2.1 技术栈

| 层次 | 技术选择 |
|------|----------|
| UI框架 | SwiftUI |
| 异步处理 | Swift Concurrency (async/await, Actor) |
| 本地存储 | Core Data + NSPersistentCloudKitContainer |
| 云同步 | CloudKit (iCloud) |
| 图表 | Swift Charts |
| 条码扫描 | AVFoundation |
| AI食物识别 | Vision Framework + Core ML |
| 健康数据 | HealthKit |
| Watch配套 | WatchKit (SwiftUI) |
| 最低版本 | iOS 17.0+ |

### 2.2 架构模式

采用 **MVVM + Repository** 模式：

```
View (SwiftUI) → ViewModel (ObservableObject) → Repository (数据抽象层) → Core Data / HealthKit
```

- **View**：纯UI，无业务逻辑
- **ViewModel**：状态管理、业务逻辑、数据转换
- **Repository**：统一数据访问，屏蔽Core Data / HealthKit / CloudKit细节
- **Model**：Core Data Entity + Value Objects

### 2.3 数据同步策略

- Core Data使用`NSPersistentCloudKitContainer`自动iCloud同步
- HealthKit数据为只读同步（从HealthKit拉取，不写入）
- 冲突解决策略：最后写入胜出（Last Write Wins）
- 同步状态指示器：设置页显示iCloud同步状态

### 2.4 食物数据库策略

- 核心1000种常见食物内置在App包中
- 剩余食物数据（10万+）首次使用时按需下载
- 下载后缓存在本地，避免重复下载
- 支持离线使用已下载的数据

### 3.3 模块2：运动追踪

#### 运动记录

**手动记录流程**
1. 点击「+」或快捷按钮
2. 选择运动类型（100+预设：有氧/力量/柔韧/球类/水上/冬季运动等）
3. 输入运动参数：
   - 有氧：时长、距离（可选）、心率区间（可选）
   - 力量（详见下方「力量训练详细记录」）
4. 自动计算消耗卡路里（基于MET值 + 用户体重）
5. 保存记录

**力量训练详细记录**

力量训练采用「动作→组数」二级结构，一个力量训练记录可包含多个动作，每个动作包含多组数据。

动作级数据：
- 动作名称（从预设库选择或自定义）
- 预设动作包含：卧推、深蹲、硬拉、引体向上、俯卧撑、哑铃飞鸟、腿举、二头弯举、三头下压、划船等50+动作
- 收藏：用户可收藏常用动作

每组数据：
- 重量（kg/lb，根据单位设置）
- 次数（reps）
- 组间休息时间（秒，可选，预设值：60s/90s/120s/180s）
- RPE 体感强度（1-10，可选，默认不填）
- 完成状态（已完成/跳过）

上次数据参考：
- 每组显示上次训练时同动作同组数的重量×次数，方便递增
- 个人纪录（1RM）高亮显示 🏆 标识

录入交互：
- 「+ 添加一组」按钮追加新组
- 重量和次数使用 +/− 按钮，步长：重量2.5kg/5lb、次数1
- 支持快捷输入（点击数字直接编辑）
- 支持修改上一组数据（长按编辑）
- RPE 使用1-10圆点选择器

训练总览（列表展开后显示）：
- 每个动作名称 + 组数标签
- 每组显示：组号 | 上次 | 重量 | 次数 | 休息
- 完成的组标记为绿色对勾
- 动作底部显示总容量（重量×次数的累计和）
- 个人纪录组标记 🏆

保存模板：
- 保存当前训练为模板（包含所有动作和组数设置）
- 下次训练时一键加载模板，直接填入重量和次数

**HealthKit同步**
- 自动拉取Apple Watch数据：步数、跑步/骑行距离、心率、活跃能量
- 同步频率：后台每15分钟 + 打开App时即时
- 同步状态标识：已同步/同步中/同步失败
- 用户可手动刷新同步

**运动类型库**
- 分类浏览：有氧、力量、柔韧、球类、水上、冬季、日常活动等
- 搜索：支持中英文搜索
- 收藏：常用运动类型标星
- 自定义模板：保存运动参数模板，下次一键记录

#### 运动历史

- **日历视图**：月历形式，有运动的日期显示橙色圆点，点击查看当日明细
- **列表视图**：按时间倒序，显示运动类型、时长、消耗卡路里
- **筛选**：按运动类型、日期范围筛选
- **搜索**：按运动名称搜索
- **编辑/删除**：左滑操作

#### 运动统计

- 本周/本月运动总时长、总消耗、平均心率
- 运动类型分布饼图
- 周/月运动频率折线图
- 个人记录（最长距离、最重举重等）

### 3.4 模块3：饮食管理

#### 食物记录

**条形码扫描**
- 实时摄像头扫描（AVFoundation）
- 查询内置食物数据库
- 匹配成功：显示食物名称、品牌、营养信息，一键添加
- 匹配失败：引导手动录入或拍照识别

**AI拍照识别**
- 拍摄食物照片
- Vision Framework + Core ML模型分析
- 识别食物品类（如「红烧肉」「白米饭」「清炒西兰花」）
- 估算份量（小碗/中碗/大碗 或 克数估算）
- 用户确认/调整品种和份量
- 计算营养信息并保存

**数据库搜索**
- 10万+食物数据库（中英文支持）
- 核心食物内置，其余按需下载
- 搜索结果按相关度排序
- 每个食物显示：名称、品牌（如有）、每100g营养数据
- 常用食物自动标记
- 支持模糊搜索和拼音搜索

**自定义食物**
- 手动录入食物名称和营养信息（每100g）：热量、蛋白质、脂肪、碳水、纤维、钠
- 保存为「我的食物」，下次快速选择
- 支持创建组合食物（如「我的早餐组合」）

**最近/常吃**
- 最近7天记录过的食物快捷列表
- 最常吃的TOP 20食物
- 一键添加，自动填入上次份量

#### 营养分析

**每日营养概览**
- 卡路里摄入 vs 目标（进度条）
- 三大营养素比例饼图（碳水/蛋白质/脂肪）
- 微量元素柱状图：纤维、钠、糖、饱和脂肪
- 饮食质量评分（0-100，基于膳食指南）

**饮食详情**
- 按餐段分组：早餐、午餐、晚餐、加餐
- 每餐段的热量和营养明细
- 点击食物查看详细营养信息
- 支持修改份量和删除

**饮食趋势**
- 周/月卡路里趋势折线图
- 三大营养素摄入趋势
- 饮食评分趋势
- 与运动消耗叠加对比图（摄入 vs 消耗）

#### 水分追踪

- 快捷按钮：150ml/250ml/350ml/500ml 预设杯型
- 自定义输入毫升数
- 今日饮水进度条（目标可自定义，默认2000ml）
- 饮水历史记录
- 可设置定时提醒（间隔可配）

### 3.5 模块4：趋势与报告

**身体数据趋势**

**饮食趋势**
- 日均卡路里摄入趋势

**周报/月报**
- 自动生成周报（每周一）和月报（每月1日）

- 可在App内查看，也可导出为图片分享

## 4. 新手引导流程

### 4.1 首次启动引导（5步）

| 步骤 | 内容 | 输入 |
|------|------|------|
| 1 | 欢迎 + 基本信息 | 性别、出生日期、身高 |
| 2 | 当前身体数据 | 体重、体脂率（可选） |
| 3 | 健身目标 | 减脂/增肌/维持/塑形 |
| 4 | 活动水平 | 久坐/轻量/中等/重度（带说明图） |
| 5 | 目标设定 | 自动计算推荐值 + 可手动调整 |

引导结束后自动生成：
- 每日卡路里目标（基于BMR × 活动系数）
- 营养素比例推荐
- 运动频率建议

### 4.2 渐进式功能解锁

- 新用户前3天：简化界面，只显示核心数据
- 3天后：解锁趋势图表
- 7天后：解锁高级分析和智能建议

## 5. Apple Watch配套功能

### 5.1 Watch App

- 快速记录运动：选择类型 → 开始计时 → 结束保存
- 快速记录饮水：点按杯型图标
- 快速称重：输入体重
- 今日进度：三个环形（卡路里/运动/饮水）

### 5.2 Complications

- 环形进度（Modular/Utility脸）
- 今日卡路里余额（Simple脸）
- 下次称重提醒时间

### 5.3 交互设计

- Digital Crown调节数值
- 侧边按钮快速切换记录类型
- 触觉反馈：达标时轻震动确认

## 6. 非功能性需求

### 6.1 性能

| 指标 | 要求 |
|------|------|
| 冷启动时间 | < 2秒 |
| 食物搜索响应 | < 200ms |
| AI识别响应 | < 3秒 |
| 内存占用 | < 150MB |
| Core Data查询 | < 50ms |

### 6.2 可靠性

- 所有本地数据操作原子性保证
- HealthKit同步失败自动重试3次
- CloudKit同步冲突：Last Write Wins
- 数据库自动迁移（Core Data Lightweight Migration）

### 6.3 隐私

- 无第三方数据上传
- HealthKit数据仅读取，不写入
- 相机权限仅用于条码扫描和食物拍照
- 相册权限仅用于选择食物照片
- 通知权限用于提醒（可选）
- 所有权限请求在用户首次触发功能时弹出，不提前索要

## 7. 数据模型概览

### 核心实体

```
User (用户配置)
├── 目标值（体重、卡路里、运动时长等）
├── 个人资料（性别、身高、出生日期等）
└── 偏好设置（单位、主题、语言等）

Workout (运动记录)
├── 类型、时长、距离
├── 消耗卡路里
├── 心率数据（如有）
├── 力量训练分组（如有）
│   └── Exercise (动作)
│       ├── 动作名称
│       ├── 收藏标记
│       └── ExerciseSet (组数据)
│           ├── 组号
│           ├── 重量（kg/lb）
│           ├── 次数（reps）
│           ├── 组间休息时间（秒）
│           ├── RPE体感强度（1-10，可选）
│           └── 完成状态（completed/skipped）
├── 数据来源（手动/HealthKit）
└── 模板引用（如有）

FoodEntry (饮食记录)
├── 食物引用
├── 份量
├── 餐段（早/午/晚/加餐）
└── 记录时间

Food (食物数据库)
├── 名称、品牌
├── 每100g营养数据（热量、蛋白质、脂肪、碳水等）
└── 条形码（如有）

BodyMeasurement (身体数据)
├── 体重、体脂率
├── 肌肉量、腰围（可选）
└── 记录时间

WaterEntry (饮水记录)
├── 毫升数
└── 记录时间
```
